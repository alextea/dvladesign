    DVLA design elements

    Vehicle registration marks

    Usage: Where the mark is presented to the user, outside of longer copy, consider presenting the mark styled like a physical number plate.

    Benefits: User recognition is high, the distinctive style allows it to be presented without a label telling the user it is a vehicle registration mark.

    Implementation: The mark looks best when presented at medium to large headline sizes:

    CU57 ABD 300%

    CU57 ABC 200%

    CU57 ABC 150% CU57 ABC 120%

    A class .reg-mark is available for inclusion in markup. The border width and rounded corner are measured using em units, so they scale with the font size.

    The UK number plate font is available as a base64 encoded webfont and can be added to pages that need it using:

    Numberplate Font encoded as base64 in a CSS file

    And where it's needed in your HTML use:
    <link href="/public/stylesheets/numberplate.css" media="screen" rel="stylesheet" type="text/css" /> or just include in your main CSS styles.

    Fallback to New Transport or sans is acceptable.

    User information playback

    Usage: DVLA services have a strong preference for playing back details confirmed by a user during certain processes.

    These information playback areas often have the opportunity to go back, or correct details that are wrong.

    Some services use a block of playback information at the top of a page, other services use a block of playback information on the right hand side of the page.

    Form & document pictograms

    Usage: show a stylised illustration of a document where a service refers to one the user may need to complete a transaction.

    Benefits: user research has consistently found that these are very useful for people who need to reference a paper document. The forms usually have a number code, but a quick visual check can be quicker in locating it, and a specific bit of information (eg. reference number) on it.
